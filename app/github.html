Estructura del proyecto según la imagen

El proyecto tiene una estructura organizada para un sitio web de varias páginas:

taller3/
├── app/
│   ├── forms.html
│   ├── tables.html
│   └── github.html
├── assets/
│   ├── css/
│   ├── img/
│   └── js/
├── index.html
└── README.md
Elemento Descripción
app/registro.html Página destinada a formularios.
app/tables.html Página destinada a tablas.
app/github.html Página destinada a la información guía para el manejo en GitHub.
assets/css Carpeta para archivos de estilos CSS, si el proyecto los requiere.
assets/img Carpeta para imágenes del proyecto.
assets/js Carpeta para archivos JavaScript.
README.md Archivo README con documentación.
index.html Página principal del proyecto.

nota: esta guia es solo bootstrap para diseño Visual

Requisitos iniciales
  • Tener una cuenta activa en GitHub.
  • Tener instalado Git en el computador.
  • Tener instalado Visual Studio Code.
Configuración de Git desde Visual Studio Code

Abre la terminal integrada de Visual Studio Code:

Menú: Terminal → New Terminal

Configura tu nombre y correo:

git config --global user.name "Tu Nombre"
git config --global user.email "tu_correo@email.com"
Inicializar Git en el proyecto

Ubícate en la carpeta raíz del proyecto y ejecuta:

git init
git add .
git commit -m "Primer commit"
📌 Este primer commit guarda la versión inicial del proyecto.
Crear repositorio en GitHub
  1. Ingresar a GitHub.
  2. Hacer clic en New repository.
  3. Asignar un nombre al repositorio.
  4. seleccionar si sera publico o privado.
  5. crear el repositorio sin agregar README si el proyecto ya tiene archivos locales.
Conectar proyecto local con GitHub

Copia la URL del repositorio creado y ejecuta:

git remote add origin https://github.com/usuario/proyecto-bootstrap-adso.git
git branch -M main
git push -u origin main
Cambia usuario y proyecto-bootstrap-adso por los datos reales del repositorio.
Manejo de ramas: main, dev y feature
main

Contiene la versión estable del proyecto lista para producción.

dev

Contiene los cambios globales en desarrollo activo.

feature

Se usa exclusivamente para programar funciones aisladas.
